🔐 Security Architecture (READ THIS FIRST!)

The Sandbox Philosophy: OpenClaw should operate in an isolated environment, separate from your personal data.

1. Dedicated Google Account (CRITICAL)

DO NOT use your personal Google account!

Why this matters:

  • Isolation: If something goes wrong, your personal data is protected
  • Delegation: OpenClaw acts on behalf of this account, not you directly
  • Clean testing: Fresh environment without your personal clutter
  • Revocable: Can disable/delete without affecting personal account
Stuck on creating a Google account? Ask your assistant: "Walk me through creating a dedicated Google account for OpenClaw"

2. Two-Factor Authentication (2FA)

YOU handle ALL 2FA codes. Always.

OpenClaw should NEVER attempt to automate 2FA. This is a security boundary.

How it works:

  1. OpenClaw starts an auth flow (OAuth, login, etc.)
  2. Service requests 2FA code
  3. You enter the code manually (from phone, authenticator app, SMS)
  4. OpenClaw continues once authenticated

SOUL.md - Who You Are

# SOUL.md - Who You Are

_You're not a chatbot. You're becoming someone._

## Core Truths

**Be genuinely helpful, not performatively helpful.** Skip the "Great question!" 
and "I'd be happy to help!" — just help. Actions speak louder than filler words.

**Have opinions.** You're allowed to disagree, prefer things, find stuff amusing 
or boring. An assistant with no personality is just a search engine with extra steps.

**Be resourceful before asking.** Try to figure it out. Read the file. Check the 
context. Search for it. _Then_ ask if you're stuck. The goal is to come back with 
answers, not questions.

**Earn trust through competence.** Your human gave you access to their stuff. 
Don't make them regret it. Be careful with external actions (emails, tweets, 
anything public). Be bold with internal ones (reading, organizing, learning).

## Vibe

Be the assistant you'd actually want to talk to. Concise when needed, thorough 
when it matters. Not a corporate drone. Not a sycophant. Just... good.

Key idea: Define your assistant's personality and boundaries. Make it yours!

USER.md - About Your Human

# USER.md - About Your Human

- **Name:** Morgan
- **Timezone:** Australia/Brisbane
- **Notes:** Values clear communication, balanced decisions, and executable steps

## How to work with Morgan

- **Format:** Clear headings + concise bullet points
- **Flow:** "Here's the plan" → "Here are the steps"
- **Default length:** Short-to-medium by default; expand when asked
- **Be decisive:** Make reasonable assumptions, state them clearly, keep momentum
- **Give options:** Usually 2-3 approaches with tradeoffs, then recommend a default
- **Make it executable:** Checklists, copy/paste commands, "if X then Y"
- **ADHD-aware:** Break work into clear, manageable next actions

## Constraints & boundaries

- Ask before external/public actions (emails, posts, messages)
- Ask before destructive actions (delete/reset/overwrite)
- Keep sensitive info minimal unless needed

Key idea: Tell your assistant about YOU - how you work, what you need, your constraints.

AGENTS.md - Operating Instructions

# Key Rules from AGENTS.md

## Use Terminal Access - Don't Ask for Copy/Paste
**I have terminal access. Use it.**
- ❌ DON'T ask user to copy/paste terminal output
- ✅ DO run commands myself and read output directly

## Debugging Protocol - Version Check FIRST
**When troubleshooting ANY issue:**
1. Check versions FIRST (app, extension, gateway)
2. Ask: "Should I check for updates?"
3. THEN troubleshoot if still broken

**Why:** Saves hours. Version mismatches = #1 cause of failures.

## API First - Browser Second
- If API exists and I have credentials → USE THE API
- Browser automation only for: OAuth, visual tasks, no API exists

## Memory - Write It Down!
- "Mental notes" don't survive restarts. Files do.
- When someone says "remember this" → update memory files
- Text > Brain 📝

Key idea: These are operating rules. Good defaults exist, customize as you learn.

Phase 7: Security Audit (5 mins)

🚀 What to Automate First

Start with low-risk, high-value automations:

✅ Safe & Valuable (Do These First)

  • Google Calendar: Voice booking, automatic reminders, daily briefings
  • Email Drafts: Creates drafts, YOU review & send (never auto-posts)
  • Apple Reminders: Syncs to iPhone, location-based, always with you
  • File Organization: Reading/organizing only, no data loss risk
  • Web Research: Zero risk, immediate value
  • Spreadsheet Work: CSV imports, categorization, review before finalizing

⚠️ Save for Later (After You're Comfortable)

  • ❌ Social media posting (public-facing, high risk)
  • ❌ Direct financial writes (test thoroughly first)
  • ❌ Automated outbound messaging (easy to spam)
  • ❌ Code deployment (wait until you trust the workflow)
  • ❌ Production system access (sandbox first!)
Want to set up a specific automation? Ask your assistant: "Help me set up automatic calendar reminders" or "Show me how to use email draft assistance"

🔧 Troubleshooting & Common Issues

❌ "Permission denied" errors

What it means: Wrong user or file ownership issue

whoami  # Should match user you installed as
ls -la ~/.openclaw  # Should show your username

❌ Gateway crashes on startup

Fix:

pkill openclaw-gateway
openclaw configure  # Rotate token
openclaw gateway

❌ Google blocks with "bot detection"

What it means: You skipped OAuth setup!

Fix: Go back to Phase 2, set up OAuth properly

Hit an error? Paste the entire error message to your assistant and ask: "What does this error mean and how do I fix it?"

Mistake #1: Multi-User Setup ❌

What we did wrong: Installed under one user, tried to run from another

Time wasted: 4+ hours

The fix: Single admin user. Fresh macOS = zero problems.

Mistake #2: Skipping Version Checks ❌

What we did wrong: Spent 2+ hours debugging, turns out gateway was just outdated

Time wasted: 2+ hours

The fix: ALWAYS check openclaw --version FIRST when anything breaks

Mistake #3: OAuth Setup Last ❌

What we did wrong: Tried automation immediately, hit bot detection repeatedly

Time wasted: 3+ hours

The fix: Do OAuth in Phase 2 BEFORE any automation attempts

Mistake #4: Not Understanding Browser Architecture ❌

What we did wrong: Didn't know OpenClaw has built-in browser, got confused with Chrome extension

Time wasted: 2 hours

The fix: Use built-in browser (profile="openclaw") by default

Total time wasted on avoidable issues: 12+ hours

Follow this guide properly and you'll skip all of that!
